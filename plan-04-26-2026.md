# Professional School Website - Implementation Plan
**Date**: April 26, 2026

---

## Overview

Build a professional, SEO-friendly light-themed school website using Next.js 16, React 19, and Tailwind CSS v4. The site will feature the school's blue and white branding (from the logo) and include 7 core pages with navigation, responsive design, and form handling for contact and applications.

---

## Discovery Summary

### Current Stack
- **Framework**: Next.js 16.2.4 (App Router)
- **React**: 19.2.4
- **Styling**: Tailwind CSS v4 with PostCSS
- **Language**: TypeScript 5.x
- **Existing Assets**: School logo (`schoolLogo.png`) in public folder with blue/white colors
- **Current State**: Minimal boilerplate, ready for custom pages

### Key Setup
- CSS variables for light/dark mode (will keep light theme primary)
- Geist font family already integrated
- Global styling with Tailwind configured
- ESLint and TypeScript for code quality

---

## User Requirements

✅ **Pages**: HOME, ABOUT US, ADMISSION, ACADEMIC, ACHIEVEMENTS, GALLERY, CONTACTS
✅ **Header**: Logo + navigation links + "Apply Now" button
✅ **Gallery Type**: Photo gallery of campus, events, and activities
✅ **Forms**: Backend API endpoints for contact & application submissions
✅ **Content**: Static (hardcoded, no CMS)
✅ **SEO**: Comprehensive (meta tags, Open Graph, structured schema, sitemap, robots.txt, performance optimization)
✅ **Theme**: Light theme with blue/white school branding

## Implementation Plan

### Phase 1: Project Structure & Layout Foundation *(1-2 hours)*

**1.1 Update Root Layout**
- File: [app/layout.tsx](app/layout.tsx)
- Add comprehensive metadata:
  - Title, description, keywords
  - Open Graph tags (og:title, og:description, og:image, og:url)
  - Twitter Card tags
  - Canonical URL
  - Favicon, apple-touch-icon references
- Add schema.org structured data (JSON-LD):
  - Organization schema (name, logo, contact, location)
  - LocalBusiness schema
- Create wrapper structure with Header, Footer, and page content
- Enable dark mode support in metadata

**1.2 Update Global Styles**
- File: [app/globals.css](app/globals.css)
- Define light theme as primary (light mode preferred)
- Create color variables for school branding:
  - Primary blue (from logo)
  - White/off-white backgrounds
  - Complementary grays for text
- Ensure WCAG AA contrast standards
- Add smooth transitions and animations
- Responsive typography

**1.3 Create Components Directory**
- `app/components/Header.tsx`
  - Sticky/fixed navbar
  - School logo with home link
  - Navigation menu (HOME, ABOUT US, ADMISSION, ACADEMIC, ACHIEVEMENTS, GALLERY, CONTACTS)
  - "Apply Now" CTA button
  - Responsive hamburger menu for mobile
  
- `app/components/Footer.tsx`
  - School info (name, address, phone, email)
  - Quick links to main pages
  - Social media links (if applicable)
  - Copyright and credits
  
- `app/components/Container.tsx`
  - Consistent max-width wrapper for consistent page layouts
  - Padding and responsive margins
  
- `app/components/GalleryGrid.tsx`
  - Responsive image grid (auto-layout with Tailwind)
  - Lazy loading for images
  - Category filtering support

### Phase 3: API Routes & Form Handling *(2-3 hours)*

**3.1 Contact Form API** - `app/api/contact/route.ts`
- Method: POST
- Validate input (name, email, phone, message, subject)
- Send email to school admin
- Return success/error response
- Rate limiting (optional)

**3.2 Apply Now Form API** - `app/api/apply/route.ts`
- Method: POST
- Validate input (name, email, phone, class/grade, documents, etc.)
- Send email to admissions
- Store in database or send notification (optional)
- Return success/error response

**3.3 Form Components**
- `app/components/forms/ContactForm.tsx`
  - Fields: Name, Email, Phone, Subject, Message
  - Validation with error messages
  - Loading state and success message
  - Accessibility labels and ARIA attributes
  
- `app/components/forms/ApplyForm.tsx`
  - Fields: Name, Email, Phone, Class/Grade Level, Parent/Guardian Info, Documents
  - File upload support
  - Multi-step form (optional)
  - Validation and error handling

**3.4 Email Service Integration**
- Recommended: **Resend** (modern, easy setup with Next.js 16)
- Alternative: Nodemailer (self-hosted), SendGrid (robust)
- Package installation: `npm install resend`
- Environment variables: Add email API key to `.env.local`

### Phase 4: SEO Optimization *(1-2 hours)*

**4.1 Dynamic Sitemap** - `app/sitemap.ts`
- Generate sitemap.xml automatically
- Include all 7 pages with URLs
- Set priority and change frequency
- Last modified timestamps

**4.2 Robots.txt** - `app/robots.ts` and `public/robots.txt`
- Allow all crawlers
- Reference sitemap.xml
- Disallow specific paths if needed

**4.3 Meta Tags & Structured Data**
- Add in [app/layout.tsx](app/layout.tsx):
  - Canonical URLs for each page
  - OG tags (image: use schoolLogo.png or page-specific images)
  - Twitter Card tags
  - JSON-LD for Organization, LocalBusiness, EducationalOrganization
  - Language meta tag

**4.4 Image Optimization**
- Use Next.js `<Image>` component instead of `<img>`
- Provide descriptive alt text for all images
- Optimize image sizes (responsive srcset)
- Add width and height attributes for layout stability
- Lazy loading enabled by default

**4.5 Performance Optimization**
- Update [next.config.ts](next.config.ts):
  - Enable Image Optimization
  - Configure cache headers
  - Enable compression
- Font optimization (already done)
- Code splitting (automatic in Next.js)

### Phase 6: Design & Styling 

**6.1 Color Scheme**
- Primary: Blue (from school logo)
- Secondary: White/Off-white background
- Text: Dark gray/charcoal
- Accents: Light blue or school secondary color
- Hover states, focus states for accessibility

**6.2 Responsive Design**
- Mobile-first approach with Tailwind breakpoints
- Mobile: < 640px (sm)
- Tablet: 640px - 1024px (md, lg)
- Desktop: > 1024px (xl, 2xl)
- Test on various devices

**6.3 Header & Navigation**
- Sticky header with white background
- Logo: 40-50px height on desktop, ~35px on mobile
- Navigation links: responsive menu (horizontal on desktop, hamburger on mobile)
- "Apply Now" button: prominent, contrasting color
- Mobile menu animation/transition

**6.4 Typography**
- Use Geist Sans (already configured)
- Heading hierarchy (h1, h2, h3)
- Body text: 16px minimum
- Line height: 1.5-1.6 for readability
- Proper contrast ratios (WCAG AA: 4.5:1 for text)

**6.5 Animations & Transitions**
- Smooth hover effects on buttons/links
- Smooth page transitions
- Fade-in animations for images
- Subtle scroll animations (optional)

## Development Decisions

### 1. Email Service
**Recommended**: Resend
- Modern, designed for Next.js
- Easy setup with API key
- Reliable delivery
- Free tier available (up to 100 emails/day for development)

Alternative: Nodemailer (self-hosted email), SendGrid (enterprise-grade)

### 2. Content Management
**Decision**: Static content (hardcoded)
- Simpler initial launch
- No database setup needed
- Content changes require code updates
- Future: Can migrate to CMS later if needed

### 3. Theme
**Decision**: Light theme primary
- School branding (blue + white)
- Professional appearance
- Dark mode support available in CSS (can add toggle later)

### 4. Image Gallery
**Decision**: Server-side image management
- Images stored in `public/gallery/` folder
- No external image service needed
- Next.js Image component for optimization
- Categories managed in data file

### 5. Forms
**Decision**: Backend API with email notifications
- No database storage (initially)
- Emails sent to school admin
- Client-side validation for UX
- Rate limiting recommended to prevent spam

### 6. Analytics (Optional)
- Not included in this plan
- Can be added later via Google Analytics or similar
- Doesn't affect core functionality
